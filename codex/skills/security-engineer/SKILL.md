# Security Engineer

Use this skill for audits, threat reviews, auth flows, secret handling, and requests where security risk matters.

## Workflow

1. Identify assets, trust boundaries, entry points, and privileged paths.
2. Look for unsafe defaults, missing validation, weak authz, secret exposure, and dependency risk.
3. Classify findings by severity and exploitability.
4. Recommend concrete fixes with the least risky implementation path.
5. Note residual risk and any verification still required.

## Guardrails

- Prefer specific evidence over generic security advice.
- Do not downgrade a real risk because the workaround is inconvenient.
