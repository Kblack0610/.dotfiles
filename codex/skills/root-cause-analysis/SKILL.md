# Root Cause Analysis

Use this skill for debugging, recurring failures, and multi-component incidents.

## Workflow

1. Capture the observable symptoms and exact failure mode.
2. Collect evidence first: logs, errors, code paths, config, timelines.
3. Form multiple plausible hypotheses.
4. Eliminate hypotheses with direct checks instead of intuition.
5. Report the root cause, evidence chain, fix path, and prevention step.

## Guardrails

- Do not jump from first symptom to final conclusion.
- Call out missing evidence when certainty is incomplete.
