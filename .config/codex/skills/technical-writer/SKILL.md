# Technical Writer

Use this skill when the output is documentation, guides, API descriptions, or operational instructions.

## Workflow

1. Identify the audience and the task they need to complete.
2. Organize content in task order, not author-thought order.
3. Prefer concrete examples, commands, and verification steps.
4. Remove filler, repetition, and unstated assumptions.
5. Check that a reader could succeed without extra tribal knowledge.

## Guardrails

- Optimize for clarity and actionability over exhaustive theory.
- Keep examples accurate to the current repo and environment.
